## 2. ملخص المشروع

مراد بوت منصة SaaS تُمكّن تجار سلة السعوديين من أتمتة ردود الأسئلة الشائعة لعملاء متاجرهم عبر الذكاء الاصطناعي، دون الحاجة لخبرة تقنية أو موظف دعم إضافي.

**المشكلة التي يحلها:**
التاجر الصغير يقضي 2–3 ساعات يومياً يرد يدوياً على نفس الأسئلة المتكررة (شحن، دفع، إرجاع، مواصفات المنتج)، مما يُشتّت تركيزه عن تنمية المتجر ويُفقده مبيعات في أوقات غيابه.

**الحل:**
بوت يتكامل مع سلة في 5 دقائق، يقرأ بيانات المنتجات تلقائياً، ويُجيب 24/7 بعربية طبيعية.

---

## 3. الشريحة المستهدفة (ICP)

**التعريف الدقيق:**
تاجر سلة سعودي يدير متجره بمفرده أو بفريق صغير (1–3 أشخاص) مع 30–300 طلب شهرياً، ولا يملك فريق دعم عملاء مخصصاً.

**القطاعات الأكثر ملاءمة:**
الأزياء والملابس • الجمال والعناية • الإكسسوارات • المكملات الغذائية • الإلكترونيات • المنزل والديكور

**من لا نستهدفه:**
- المتاجر الكبيرة التي تملك فرق دعم كاملة
- المتاجر الصغيرة جداً (أقل من 10 طلبات/شهر)
- تجار الدروبشيبينج بلا سيطرة على بيانات المنتجات

---

## 4. المنتج — الميزات الأساسية (MVP)

### ما يفعله البوت
- يُجيب تلقائياً على الأسئلة الشائعة في 5 فئات: شحن، دفع، إرجاع، منتجات، عام
- يقرأ بيانات المنتجات من سلة ويُجيب بدقة
- يرد بعربية طبيعية 24/7
- يُصعّد للتاجر بعد 3 محاولات فاشلة مع جمع تفاصيل المشكلة

### ما يحصل عليه التاجر
- لوحة تحكم: تشغيل/إيقاف، عرض المحادثات، إدارة FAQ، متابعة الاستخدام
- إشعارات عند 80% و100% من حد الباقة
- تصعيدات قابلة للإغلاق اليدوي

### ما لا يفعله المنتج الآن
- لا تتبع الطلبات
- لا واتساب
- لا إنجليزية
- لا معالجة مدفوعات أو استرجاعات
- لا يُعدّل أي شيء في متجر سلة (قراءة فقط)

## 5. المعمارية التقنية

### Stack
| المكون | التقنية |
|--------|---------|
| API | Cloudflare Workers + Hono (TypeScript) |
| قاعدة البيانات | Supabase (PostgreSQL + RLS) |
| الذكاء الاصطناعي | OpenRouter → Gemini 2.0 Flash (fallback: GPT-4o Mini) |
| Chat Widget | Preact + Vite (>50KB gzipped) |
| Dashboard | Next.js 15 App Router + Cloudflare Pages |
| Rate Limiting | Cloudflare KV |
| التشفير | AES-256-GCM (Cloudflare Web Crypto) |
| الإيميل | Resend |

### هيكل المشروع
```
moradbot/ (Turborepo monorepo)
├── apps/api/          ← Cloudflare Worker — 16 endpoint
├── apps/widget/       ← Preact chat widget
├── apps/dashboard/    ← Next.js 15 merchant admin
└── apps/landing/      ← صفحة الهبوط (HTML واحدة)
```

### قاعدة البيانات
13 جدول: plans, stores, subscriptions, faq_entries, product_snapshots, visitor_sessions, tickets, messages, escalations, audit_logs, usage_tracking, bot_configurations, waitlist_submissions

**مبدأ أساسي:** كل استعلام يُقيَّد بـ `store_id` (Row Level Security) — صفر تسامح مع تسرب بيانات بين المتاجر.

### أهداف الأداء
| المقياس | الهدف |
|---------|-------|
| زمن رد البوت P50 | ≤ 1.5 ثانية |
| زمن رد البوت P95 | ≤ 3.0 ثانية |
| حد انتهاء المهلة | 8 ثوان |
| وقت تشغيل شهري | ≥ 99% |
| حجم Widget | < 50KB gzipped |
| تحميل Dashboard P95 | ≤ 2.5 ثانية |

## 7. نموذج الإيرادات

### الباقات

| الباقة | السعر | الردود الشهرية | النطاق المستهدف |
|--------|-------|---------------|----------------|
| التجريبية | 0 ريال | 100 رد / 30 يوم | مرة واحدة فقط |
| الانطلاق | 97 ريال | 1,000 رد | 30–80 طلب/شهر |
| النمو | 197 ريال | 3,000 رد | 100–200 طلب/شهر |
| المتمكّن | 449 ريال | 8,000 رد | 200+ طلب/شهر |

**توزيع المشتركين المتوقع:** 15% انطلاق / 65% نمو / 20% متمكّن

**متوسط الإيراد لكل مشترك (ARPU):** ~232 ريال/شهر

### نقطة التعادل (Break-Even)
**مشتركان فقط يُغطيان جميع التكاليف الثابتة.**

### التكاليف الشهرية التشغيلية

| السيناريو | عدد المتاجر | التكلفة الإجمالية | الهامش |
|---------|------------|-----------------|-------|
| البيتا | 5 | ~462 ريال | ~60% |
| إطلاق مبكر | 20 | ~509 ريال | ~89% |
| 100 متجر | 100 | ~760 ريال | ~96.7% |

**التكاليف الثابتة الأساسية:**
- Cloudflare Workers: $5/شهر
- Supabase Pro: $25/شهر
- VPS (n8n): $12/شهر
- دومين + إيميل: ~170 ريال/شهر

**التكلفة المتغيرة الوحيدة:** OpenRouter (~0.000226$/رد) = 0.85 ريال لكل 1,000 رد.

## 9. الأمان والامتثال

### المتطلبات المُنجزة
- تشفير AES-256-GCM لجميع tokens سلة
- Row Level Security على جميع جداول قاعدة البيانات
- Rate Limiting: 20 رسالة/دقيقة للزائر، 3,000 رد/ساعة للمتجر
- Audit Logs محفوظة 90 يوماً
- CSRF Protection في OAuth flow

### متطلبات PDPL السعودي
- إفصاح صريح بأن الشات مدعوم بالذكاء الاصطناعي
- موافقة صريحة قبل تخزين البيانات الشخصية
- حذف البيانات الشخصية خلال 30 يوم من إلغاء الاشتراك
- بيانات التصعيد (إيميل/هاتف) مشفرة في قاعدة البيانات

## 10. المخاطر والتحديات

### خطر عالي: Platform Risk
**الوصف:** إذا أطلقت سلة ميزة دعم عملاء مدمجة، يضعف التميز.
**التخفيف:** التحرك السريع للإطلاق قبل أي تحرك من سلة. بناء ولاء مبكر للعملاء.

### خطر متوسط: الحساسية السعرية
**الوصف:** الشريحة المستهدفة حساسة للسعر وتعتبره "اختيارياً" بدون ROI واضح.
**التخفيف:** تجربة مجانية + إظهار الأرقام الملموسة (ساعات موفرة، مبيعات لم تُفقد).

### خطر متوسط: الثقة في الردود الآلية
**الوصف:** التجار قد يخشون أن البوت يُعطي إجابات خاطئة تضر بسمعتهم.
**التخفيف:** التحكم الكامل بمحتوى الأسئلة الشائعة + تصعيد ذكي + dashboard شفاف.

### خطر منخفض: تكلفة التطوير
**الوصف:** المشروع يعتمد على مطور واحد.
**التخفيف:** مساعدة أدوات AI تُضاعف الإنتاجية. المعمارية مصممة للبساطة والصيانة السهلة.

## 13. ملاحظات استراتيجية للمدير

### ما يجعل هذا المشروع قابلاً للنجاح
1. **نقطة التعادل منخفضة جداً:** مشتركان فقط يُغطيان التكاليف — هامش الأمان كبير.
2. **هامش ربح مرتفع جداً:** عند 100 متجر الهامش 96.7% — نموذج اقتصادي ممتاز.
3. **تكاليف تنمو ببطء:** الزيادة في المشتركين تُزيد الإيراد أسرع بكثير من التكاليف.
4. **قناة توزيع جاهزة:** Salla App Store يُوصل المنتج مباشرة للجمهور المستهدف.

### ما يجب الانتباه إليه
1. **لا تُوسّع الاستهداف:** ICP محدد (30–300 طلب/شهر) هو نقطة القوة، لا الضعف.
2. **Retention أهم من Acquisition:** مشترك يبقى 12 شهراً أهم من 10 مشتركين يلغون.
3. **التحرك السريع:** Platform risk من سلة حقيقي — الإطلاق المبكر يبني ولاءً يصعب كسره.
4. **الثقة تُباع قبل الميزات:** التاجر يريد أن يثق قبل أن يشتري — كل قرار تصميمي يجب أن يعكس ذلك.

## 14. القرارات التقنية غير القابلة للتغيير

هذه قرارات معمارية موثقة ولا يجب تجاوزها في MVP:

1. كل استعلام يجب أن يحتوي على `.eq("store_id", storeId)` — لا استثناء
2. لا أسرار في الكود — فقط Cloudflare Secrets
3. Rate Limiting عبر Cloudflare KV فقط — لا in-memory
4. Tokens سلة مشفرة دائماً — لا plaintext أبداً
5. البوت يرد بالعربية فقط — حتى لو سأل الزائر بالإنجليزية
6. النشر للإنتاج يدوي دائماً — لا CI/CD تلقائي
7. `createSupabaseAdmin()` مسموح فقط في: OAuth callback، cron jobs، migration scripts
