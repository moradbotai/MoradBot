## القسم الثاني: هندسة التسعير والتحكم النفسي

### 2.1 هيكل الباقات الكامل — من التجربة إلى الاحتراف

**قمع التحويل (Conversion Funnel):**

```
[جرّب مجاناً ✨]  → (30 يوم، 100 رد، لا بطاقة — مرة واحدة للمتجر)
        ↓
[الانطلاق 🚀]    → (97 ريال، 1,000 رد)  ← ~15% من المشتركين
        ↓
[النمو 📈]        → (197 ريال، 3,000 رد) ← ~65% من المشتركين ← الهدف
        ↓
[المتمكّن 🏆]    → (449 ريال، 8,000 رد) ← ~20% من المشتركين
```

#### الباقة التجريبية — جرّب مجاناً ✨

> لا تحتاج بطاقة بنكية. لا التزام. تنتهي تلقائياً بعد 30 يوماً. **مرة واحدة فقط للمتجر — لا تتكرر.**

| الخاصية | التفاصيل |
| -------- | -------- |
| **السعر** | **0 ريال — مجاناً تماماً** |
| **المدة** | 30 يوماً فقط — غير قابلة للتمديد أو التكرار |
| عدد الردود الإجمالية | 100 رد (للمدة كلها — مرة واحدة لكل متجر) |
| نموذج AI | أقوى نموذج مجاني متاح (`gemini-2.0-flash-exp:free`) — صفر تكلفة API |
| متاجر مرتبطة | 1 |
| لوحة التحكم | نفس لوحة التحكم الكاملة — مميزات مقفلة حسب الباقة |
| الدعم الفني | بلا دعم — FAQ فقط |
| شعار مراد بوت | **ظاهر على الواجهة + "مشغّل بواسطة مراد بوت"** — غير قابل للإزالة |
| بعد انتهاء المدة أو الردود | يتجمّد الحساب — يجب الاشتراك للاستمرار |
| قابلية التكرار | ❌ **مرة واحدة فقط لكل متجر — غير قابلة للتكرار بأي حساب** |
| **تكلفتها على مراد بوت** | ~0 ريال API (نموذج مجاني) |

**السيكولوجيا خلف التصميم:**

- **100 رد فقط، لا 1,000:** يكفي لإثبات القيمة في متجر متوسط النشاط (3–6 أسئلة/يوم)، لكنه ينتهي خلال 17–33 يوماً — فيشعر التاجر بـ "الحاجة" للاستمرار قبل انتهاء المدة أو حينها.
- **30 يوماً كاملة:** تعطي وقتاً كافياً للتعود على المنتج وبناء الاعتماد عليه قبل انتهاء التجربة.
- **شعار مراد بوت ظاهر:** يُشعر التاجر "أن شيئاً ناقصاً" ويحفّزه للترقية لإزالته وتخصيص الهوية.
- **مرة واحدة فقط:** يمنع إعادة التسجيل بحسابات جديدة للتحايل، ويُعزز الندرة الحقيقية للتجربة.
- **Foot-in-the-door:** بمجرد أن يُعدّ التاجر البوت ويرى عملاءه يتفاعلون معه — الإلغاء يصبح "خسارة" لشيء يعمل، لا مجرد إلغاء اشتراك.

#### الباقات المدفوعة — Decoy Effect

**المبدأ النفسي:** الشخصية A تتجذب للانطلاق، لكن الفجوة الكبيرة في القيمة مقابل فرق سعر صغير تدفعها للنمو. الشخصية C تنظر للمتمكّن أولاً، فتبدو النمو "صفقة ممتازة".

| الخاصية | باقة الانطلاق 🚀 | باقة النمو 📈 | باقة المتمكّن 🏆 |
| -------- | --------------- | ------------ | ---------------- |
| **السعر الشهري** | **97 ريال** | **197 ريال** | **449 ريال** |
| دورة الفوترة | شهري فقط | شهري فقط | شهري فقط |
| عدد الردود/شهر | 1,000 رد | 3,000 رد | 8,000 رد |
| متاجر مرتبطة | **1** | **1** | **1** |
| لوحة التحكم | نفس لوحة التحكم — مميزات مقفلة | نفس لوحة التحكم — مميزات مفتوحة جزئياً | نفس لوحة التحكم — كل المميزات مفتوحة |
| شعار مراد بوت | **ظاهر + "مشغّل بواسطة مراد بوت"** — غير قابل للإزالة | يمكن إزالته | يمكن إزالته |
| تقارير الأداء | ❌ مقفلة | ✅ متاحة | ✅ متاحة |
| API Access | ❌ مقفل | ❌ مقفل | ✅ متاح |
| دعم فني | بريد إلكتروني | بريد + واتساب | واتساب أولوية |
| **الرسالة النفسية** | "كافٍ للبداية، لكن محدود" | **"القيمة الأفضل بفارق كبير ⭐"** | "للمحترف الذي يريد الكامل" |
| **نسبة الاختيار المتوقعة** | ~15% | **~65%** | ~20% |

**ملاحظة لوحة التحكم الموحّدة:**
جميع الباقات تدخل نفس لوحة التحكم. التجار يرون المميزات المغلقة مع رسالة "ترقّى لباقة [X] لتفعيل هذه الميزة" — هذا يخلق شعوراً دائماً بأن "هناك المزيد" ويُسهّل الـ Upsell بصرياً.

**تفسير الطُّعم:**

- **الانطلاق 97 ريال:** 1,000 رد — كافٍ للمتجر الصغير (30–50 طلب/شهر) لكن ضيّق على المتجر النامي. العميل في متجر بـ 100+ طلب يحسب: "سأحتاج للترقية" — يجعل 197 ريال "منطقية جداً". بالإضافة: الشعار مقيّد يُزعج.
- **المتمكّن 449 ريال:** ميزانية واحدة ومتجر واحد = لكن المميزات كاملة. يجعل 197 ريال تبدو "الخيار الذكي".
- **النمو 197 ريال:** 6.5 ضعف الردود مقابل 2 ضعف السعر + تقارير كاملة — كل منطق الحساب يشير إليها.

**ترتيب العرض في صفحة التسعير (مهم جداً):**

```
[المتمكّن 🏆 — 449 ريال]      ← أولاً كـ "الأقوى"
[النمو 📈 — 197 ريال]         ← الوسط مع شارة "الأكثر شيوعاً ⭐"
[الانطلاق 🚀 — 97 ريال]      ← الأخير كـ "للبداية"
[جرّب مجاناً ✨ — 30 يوم]    ← أسفل الجميع، نص صغير واضح
```

**سياق سعر Salla التشغيلي للتجار:**
الباقة التشغيلية لسلة تُكلّف التاجر 99 ريال/شهر. هذا يضع باقة الانطلاق (97 ريال) في منطقة "مقارنة مباشرة بما يدفعه التاجر أصلاً" — رسالة قوية: "بنفس ما تدفعه لسلة، تحصل على مراد يخدم عملائك."

### 2.3 تحليل هامش الربح — جانب المنتج

**تكاليف API المتغيرة وهامش الربح:**

| الباقة | الإيراد | تكلفة OpenRouter | هامش الربح | الملاحظة |
| ------ | ------- | ---------------- | ---------- | --------- |
| جرّب مجاناً ✨ | 0 ريال | ~0 ريال (100 رد — نموذج مجاني) | — | استثمار تسويقي، صفر تكلفة API |
| الانطلاق 🚀 | 97 ريال | ~0.85 ريال (1,000 رد × $0.000226 × 3.75) | ~99.1% | |
| النمو 📈 | 197 ريال | ~2.55 ريال (3,000 رد) | ~98.7% | الهدف الرئيسي |
| المتمكّن 🏆 | 449 ريال | ~6.80 ريال (8,000 رد) | ~98.5% | |

> **الحساب:** تكلفة 1,000 رد = 1,000 × $0.000226 × 3.75 = 0.85 ريال (التكلفة المتغيرة رمزية — الهامش من الإيرادات يغطي التكاليف الثابتة).
> **الباقة المجانية كاستثمار تسويقي:** تكلفة كل مستخدم تجريبي ~0 ريال (نموذج مجاني). إذا تحوّل 20% منهم لمشتركين مدفوعين بمتوسط 197 ريال/شهر، فكل 5 تجارب (0 ريال تكلفة API) تُنتج مشترك بـ 197 ريال — عائد استثمار لا نهائي على مستوى API.

> **قابلية التوسع:** التكلفة الرئيسية هي استدعاء AI API وهي متغيرة وصغيرة. مع نمو المشتركين، تكاليف Cloudflare + Supabase تنمو بشكل هامشي جداً — بنية SaaS مثالية للتوسع.

**استراتيجية Upsell داخل المنتج:**

- عند وصول التجربة المجانية لـ 80 رد (80% من 100): تنبيه داخلي — *"تبقّى لك 20 رداً فقط — اشترك الآن واستمر بـ 1,000 رد كل شهر"*
- عند وصول الانطلاق لـ 800 رد (80% من 1,000): تنبيه داخلي — *"تبقّى لك 200 رداً — ترقّى لباقة النمو بـ 100 ريال إضافية فقط واحصل على 3× الردود"*
- عند وصول النمو لـ 2,400 رد (80% من 3,000): تنبيه داخلي — *"متجرك نشط جداً — باقة المتمكّن تعطيك 8,000 رد بـ 449 ريال فقط"*

## القسم الثالث: بناء العرض الذي لا يُقاوم

### 3.1 قدم في الباب — Foot-in-the-door

**الالتزام المبدئي:** باقة **جرّب مجاناً** ✨ — 30 يوماً، 100 رداً، بدون بطاقة بنكية. **مرة واحدة فقط لكل متجر.**

**لماذا 100 رد وليس أكثر؟**

في متجر بـ 30–80 طلب شهرياً، يحدث عادةً 3–6 أسئلة عملاء يومياً. هذا يعني أن الـ 100 رداً ستنتهي خلال 17–33 يوماً — قريباً من أو بُعيد نهاية الـ 30 يوم. التاجر يجد نفسه في لحظة واحدة من اثنتين: إما "الردود انتهت والمدة باقية" أو "المدة انتهت والردود على وشك النفاد" — كلتاهما لحظة تحويل مثالية. الـ 30 يوماً تُعطيه وقتاً كافياً ليتعلم ويعتمد على المنتج.

**التسلسل الكامل للتجربة المجانية:**

```
تسجيل مجاني (بريد إلكتروني فقط — لا بطاقة)
    → تفعيل البوت خلال 5 دقائق
    → [بريد اليوم 1]: "مراد جاهز — إليك 3 أسئلة جاهزة لتبدأ بها"
    → [بريد اليوم 3]: "البوت رد على X رسائل بدلاً منك 🎉"
    → [تنبيه داخلي عند 80 رد (80%)]: "تبقّى لك 20 رداً — اشترك واستمر"
    → [بريد عند نفاد الردود]: "مراد توقف — هل تريد أن يعود؟ 97 ريال/شهر (1,000 رد/شهر)"
    → [بريد اليوم 25 إذا لم تنفد الردود]: "تبقّى 5 أيام — مراد مستمر معك؟"
    → [اليوم 30]: يتجمّد الحساب، تُعرض صفحة الاختيار
    → ⚠️ لا إمكانية لإعادة التسجيل بنفس المتجر
```

**آلية Loss Aversion (نفور الخسارة):**

بمجرد أن يرى التاجر عملاءه يتفاعلون مع البوت، يصبح الإلغاء "خسارة شيء يعمل" لا مجرد "عدم شراء شيء جديد". النفس البشرية تتألم من الخسارة أكثر من سعادتها بالمكسب المساوي — هذا بالضبط ما تفعله التجربة المجانية المحدودة.

### 4.2 أنسنة البراند — الوجه البشري

**المشكلة:** البوتات والبرمجيات تبدو باردة وغير موثوقة. الحل: إظهار من يقف خلف مراد بوت والتواصل الإنساني الحقيقي.

**استراتيجية الأنسنة الأسبوعية:**

1. **"خلف الكواليس":** فيديو 15–30 ثانية أسبوعياً يُظهر فرد من الفريق يشرح قراراً أو يحل مشكلة.
2. **"قصة مراد" الشهرية:** "كنا ندير متجراً على سلة وتعبنا من الرد على نفس الأسئلة..." — بأسلوب صادق.
3. **اسم الشخصية:** "مراد" — اسم عربي شخصي، وليس "AI Assistant". هذا وحده يُضفي طابعاً إنسانياً.
4. **التعليقات:** "أهلاً، أنا سارة من فريق مراد — سأساعدك..." بدلاً من الرد العام.

### 4.3 المؤثرون الصغار — Micro-Influencer UGC

**المعيار المستهدف:**

```
متابعون: 5,000–50,000
Engagement Rate: أعلى من 4%
التخصص: تاجر/ة إلكتروني/ة سعودي/ة فعلي/ة على سلة
```

**لماذا Micro وليس Macro؟**

المؤثر الصغير يمتلك ثقة أعلى بكثير في متابعيه لأن جمهوره يشبهه ويعرفه. محتواه يبدو كـ "توصية صديق" لا "إعلان". تكلفته 200–800 ريال/مقطع مقارنة بـ 5,000–50,000 للمؤثر الكبير.

**آلية التنفيذ:**

| المرحلة | الخطوة | التفاصيل |
| ------- | ------ | -------- |
| 1 | الاستهداف | ابحث في `#متجر_سلة` عن تجار نشطين بمتابعين حقيقيين |
| 2 | التواصل | "أرسلنا لك تجربة مجانية 30 يوماً — نريد رأيك الصادق فقط" |
| 3 | المحتوى | لا script. اطلب "قبل وبعد" باستخدام حقيقي |
| 4 | الإذن | احصل على إذن لتحويله لإعلان مدفوع (Whitelisted Ad) |
| 5 | التضخيم | UGC أصيل يتفوق على الإعلان التقليدي بـ 3× |

**النص التوجيهي للمؤثر (لا script، بل توجيه):**

> "صوّر كيف تستخدم مراد بوت في متجرك بشكل طبيعي. أرنا المشكلة التي كانت تواجهك قبله، وكيف حلها. لا نريد إعلاناً — نريد قصتك الحقيقية."

## القسم الثامن: مؤشرات الأداء وإدارة الفشل والنجاح

### 8.1 المؤشرات الثلاثة الجوهرية

| المؤشر | المعيار المقبول | علامة النجاح → الإجراء | علامة الفشل → الإجراء |
| ------- | --------------- | ---------------------- | --------------------- |
| **Trial-to-Paid Rate** (تجربة → اشتراك) | أعلى من 15% | أعلى من 25% → المنتج يبيع نفسه، كرر وزد الإنفاق | أقل من 8% → مشكلة في Onboarding، اتصل بالمجرّبين شخصياً |
| **CAC** (تكلفة اكتساب مشترك) | أقل من 100 ريال | أقل من 60 ريال → ضاعف الميزانية فوراً | أكثر من 150 ريال → توقف وحلّل الاستهداف |
| **MRR Churn** (إلغاء الاشتراك الشهري) | أقل من 5%/شهر | أقل من 2% → العملاء راضون جداً، ركز على الاكتساب | أكثر من 8% → مشكلة في المنتج، أصلح قبل الاستمرار في الإنفاق |

---

### 8.2 تشخيص الفشل — جدول التحليل والتصحيح

| المشكلة | العلامة | التشخيص | التصحيح |
| -------- | ------- | -------- | -------- |
| استهداف خاطئ | CTR < 1% مع impressions عالية | الإعلان يصل للناس الخطأ | ضيّق لـ "أصحاب متاجر Salla" واختبر Hooks مختلفة |
| محتوى لا يتحول | CTR +2% لكن تسجيل < 2% | صفحة الهبوط ضعيفة | اختبر عروضاً مختلفة، بسّط الصفحة |
| تجربة بلا اشتراك | تسجيلات عالية لكن استخدام < 10% | Onboarding ضعيف أو معقد | اتصل بالمجرّبين شخصياً، أصلح نقطة الاحتكاك |
| إلغاء مرتفع | Churn > 8%/شهر | مشكلة في المنتج أو القيمة المُسلَّمة | استطلع المُلغين، أصلح قبل الاستمرار في الإنفاق |

### 8.3 إدارة إرهاق الإعلان (Ad Fatigue)

**جدول التدوير الشهري:**

| الشهر | الإجراء | السبب |
| ----- | ------- | ----- |
| 1 | اختبر Hooks A+B+C+D بالتساوي | اكتشف الفائز |
| 2 | ضاعف الفائز + أضف نسختين جديدتين | منع Fatigue المبكر |
| 3 | حوّل UGC من المؤثرين لإعلانات | المحتوى الأصيل ينعش الأداء |
| 4 | غيّر زاوية الرسالة (من "وفّر وقتك" لـ "زد مبيعاتك") | حياة جديدة للحملة |
| 5+ | دوّر: مشكلة / فائدة / قصة نجاح / مقارنة | منع الاعتياد التام |

**علامات Ad Fatigue:**

```
- انخفاض CTR أكثر من 30% خلال أسبوعين
- ارتفاع CPM مع ثبات الإنفاق
- Frequency أعلى من 5×/شخص/أسبوع
```

**الحل الفوري:**

```
1. أوقف الإعلان المُرهق فوراً
2. شغّل نسخة جديدة بـ Hook مختلف كلياً
3. وسّع الجمهور بـ 20%
4. استرح أسبوعين ثم أعد الأصلي — سيعمل مجدداً
```
